java中Action层、Service层和Dao层的功能区分
发表于 2020-03-01| 更新于 2020-03-01|后台
阅读量:

Action/Service/DAO简介:

Action: 是管理业务（Service）调度和管理跳转的。

Service: 是管理具体的功能，具体实施。

Dao: 只完成增删改查

简单概述

Model层就是对应的数据库表的实体类。Dao层是使用了hibernate连接数据库、操作数据库（增删改查）。Service层：引用对应的Dao数据库操作。Action层：引用对应的Service层，在这里结合Struts的配置文件，跳转到指定的页面，当然也能接受页面传递的请求数据，也可以做些计算处理。
